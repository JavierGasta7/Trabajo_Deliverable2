Available functions

Function Author Description Recommended test data
LoginServlet Common Authenticates user against the BD and creates session with the 4 attributes. Use any of the test accounts (see above).
LogoutServlet Common Invalidates the session and redirects to login.html. Click after signing in.
BrowseClassesServlet Marco Lists available classes with filters by activity type and instructor. Shows current spots / capacity. Sign in as ana.garcia. Try filters: activity = "Yoga", or instructor id = 2 (Carlos Mendez).
BookClassServlet Marco Books a class. Validates: membership active, not already booked, spots available. Sign in as ana.garcia and book class 4 (Zumba) — should succeed.
Sign in as javier.nunez and try — should fail (membership inactive).
Sign in as ana.garcia and try class 1 — should fail (already booked).
CancelBookingServlet Marco Cancels an existing booking. Validates >2h before class start. Releases the spot. Sign in as ana.garcia and cancel booking 1 (Yoga) or 2 (Spinning).
Try with another user's booking — should be rejected.
BookingHistoryServlet Marco Full history of the logged member: confirmed, completed and cancelled bookings, with action to rate completed ones. Sign in as ana.garcia (has 2 bookings) or laura.torres (has 2 bookings).

Database: gymtrack.accdb in the project root (read with UCanAccess via JDBC).
